How to Install ?

View on GitHub

Follow these steps to set up and run your own instance of the watchStatus API. This guide will help you clone the repository, install dependencies, configure environment variables, set up the database, and start the server.

Clone the Repository

First, clone the watchStatus API repository from GitHub and navigate to the project directory: 

git clone https://github.com/proisydev/watchStatusAPI.git
cd watchStatusAPI

Install Dependencies

Install the required dependencies using npm or yarn: 

# Using npm
npm install

# Using yarn
yarn install

Configure Environment Variables

Create a .env file in the src directory by copying the example file: 

cp src/.env.example src/.env

Edit the .env file and set the following variables:

PORT The port to run the API (default: 3000)
UPTIME_ROBOT_API_KEY_MAIN Your main UptimeRobot API key
UPTIME_ROBOT_API_KEY_READ_ONLY Your read-only UptimeRobot API key
DB_HOST, DB_USER, DB_PASSWORD, DB_NAME MySQL database configuration

Set Up the Database (Optional)

To use the database features, set the DB_ACTIVE variable to true. You need to set up a MySQL database.
The API will create the necessary tables. Configure the MySQL database credentials in the .env file: 

CREATE DATABASE watchstatus;

Note: Make sure your MySQL server is running and accessible with the credentials you've provided in the .env file.

Run the Server

Start the server in development or production mode: 

# Development mode
npm run dev
# or
yarn dev

# Production mode
npm start
# or
yarn start

The server will be available at http://localhost:3000

Need More Help?

Error Handling

The API uses conventional HTTP response codes to indicate the success or failure of requests:

  • 200 Success
  • 400 Bad Request
  • 401 Unauthorized
  • 404 Not Found
  • 429 Too Many Requests
  • 500 Internal Server Error

Rate Limits

To ensure fair usage and maintain service stability, the API implements rate limiting:

  • 100 requests per 15 min per IP address
  • Rate limit headers are included in all responses
  • Exceeding the rate limit will result in a 429 Too Many Requests response
